🔄 CQRS Read/Write Separation

Command Flow (Mutations)

POST, PUT, DELETE requests always hit the primary PostgreSQL instance. If a record updates, the CommandService automatically calls the Redis Cache Manager to evict (@CacheEvict) the stale record keys, guaranteeing eventual consistency.

Query Flow (Cache-Aside)

GET requests prioritize the Redis Cache layer (@Cacheable). The system employs advanced resilient failovers: if Redis crashes or is evicted, the QueryService gracefully catches the timeout, queries PostgreSQL synchronously, and returns the response seamlessly to the client.

⚡ Fault Tolerance Architecture

  • 🛡️
    Resilience4j Circuit Breakers

    External calls (like Razorpay API interactions or synchronous Feign requests) are wrapped in Circuit Breakers. If failure rates exceed 50%, the circuit trips to OPEN, instantly rejecting requests to protect cascading thread pool exhaustion.

  • 📡
    Eureka Service Discovery

    Hardcoded IPs are non-existent. The Gateway delegates load balancing dynamically by pulling healthy instance registries from Eureka Server. Dead nodes are automatically delisted.

  • ⚙️
    Centralized Config Server

    Spring Cloud Config serves properties to all instances asynchronously at runtime, enabling zero-downtime credential rotations.

  • 📊
    Monitoring & Distributed Tracing

    Every isolated microservice automatically streams span logs to Zipkin via Micrometer. The correlation IDs (`traceId`) injected at the API Gateway allow visualizing the entire hop-by-hop journey of a request across the cluster. Performance metrics (Memory/GC) are actively scraped by Prometheus and visualized on custom Grafana dashboards to catch high latency anomalies early.
